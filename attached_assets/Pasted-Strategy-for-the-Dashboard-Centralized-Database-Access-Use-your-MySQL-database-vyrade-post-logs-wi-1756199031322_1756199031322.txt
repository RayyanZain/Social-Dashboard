Strategy for the Dashboard

Centralized Database Access

Use your MySQL database vyrade_post_logs with the three tables:

brands → brand info

social_instagram_tiktok → Instagram/TikTok automation logs

social_linkedin_twitter → LinkedIn/Twitter automation logs

Connect to this database directly from your Vibe dashboard app.

Data Aggregation

Aggregate data from both automation workflows.

Show:

Number of posts per brand

Number of posts per platform (Instagram, TikTok, LinkedIn, Twitter)

Weekly and monthly statistics

Status breakdown (Published / Failed)

Remove drafts and scheduled posts from the view (only show published).

Dashboard Components

Sidebar Navigation (ShadCN UI theme):

Dashboard Overview

Instagram/TikTok Logs

LinkedIn/Twitter Logs

Brands

Filters:

By brand

By week / month

By platform

Main Dashboard Widgets:

Total posts

Posts per platform

Posts per brand

Status charts

Latest posts table with CRUD operations

Tables & CRUD

Brands Table

Fields: id, brand_name

Operations: Create, Read, Update, Delete

Social Logs Table

Fields from social_instagram_tiktok & social_linkedin_twitter

CRUD: Update content, status; Delete incorrect entries; Add new logs manually

Charts / Data Visualization

Bar chart: Posts per brand

Line chart: Weekly / monthly posts trend

Pie chart: Platform-wise post distribution

Integration with n8n

Dashboard reads from the same MySQL database your n8n workflows are writing into.

Automatic live updates from workflows (no need to duplicate data).

Filtering allows you to check how much content was published, on which brand, on which platform, weekly/monthly.
